# TEST MODE ガイド

## 概要

TEST MODEは、PS5がなくてもダッシュボードの動作を確認できる機能です。デモデータが自動生成され、速度/RPM/ギア/ペダル/タイヤ温度/サスペンション/STEER RESPONSE（デモ操舵）などがシミュレートされます。

## 使用方法

### 1. ダッシュボードを起動

```bash
docker compose up --build
```

### 2. ブラウザでダッシュボードを開く

```
https://localhost:8080
```

> 自己署名証明書を使用しているため、初回は「この接続ではプライバシーが保護されません」等の警告が出ます。Chrome は「詳細設定」→「localhost にアクセスする（安全ではありません）」で続行してください。

### 3. TEST MODEを有効化

ヘッダーの操作メニューバー **モード → TEST MODE** を選択します（有効化されるとメニューの TEST MODE に ✓ が付きます）。

### 4. デモデータの表示を確認

以下のデータがシミュレートされます：
- 速度、RPM、ギア位置の変化
- スロットル、ブレーキ入力
- タイヤ温度の変化
- サスペンション高さの変化
- STEER RESPONSE のデモ操舵（アンダー→ニュートラル→オーバーを周期的に巡回）

### 5. TEST MODEを停止

もう一度 **モード → TEST MODE**（STOP TEST）を選択すると、デモデータの生成が停止します。

## デモ軌跡データ

TEST MODEは、スズカサーキットのレイアウトを模した20点の座標データ（`demoTrajectory`、`test-mode.js`）を内部に持っています。これは画面表示用ではなく、ラップタイムやセクター経過などテレメトリ解析ロジックを一巡させるために内部でのみ使用されます（コース推定機能自体もTEST MODEでは使用されません）。

## デモデータ仕様

### 生成されるデータ

| フィールド | 値の範囲 | 説明 |
|-----------|---------|------|
| position_x | -300～300 | コース上のX座標（スズカ） |
| position_z | -300～250 | コース上のZ座標（スズカ） |
| speed_kmh | 50～285 | コーナーごとに変化 |
| rpm | 2000～9500 | ギアに応じて変化 |
| gear | 1～6 | コーナーに応じて適切に変化 |
| throttle_pct | 0～100 | コーナーに応じて変化 |
| brake_pct | 0～80 | コーナー進入時に増加 |
| tyre_temp | 60～110 | 速度・負荷・ラップ数で変化 |
| susp_height | 60～95mm | ブレーキ・加速・空力で変化 |

### 更新頻度

デモデータは**5Hz**（200ms間隔、`constants.js` の `TEST_MODE_CONFIG.intervalMs=200`）で更新されます。スムーズなアニメーションを実現しつつ、ブラウザのパフォーマンスを維持します。

### 物理シミュレーション

デモデータには以下の物理法則が組み込まれています：

1. **タイヤ温度**: 速度、横G、ラップ数に応じて変化
2. **サスペンション**: ブレーキ時のダイブ、加速時のスクワット、空力ダウンフォースを考慮
3. **タイヤ空気圧**: 温度上昇に伴う圧力増加（理想気体の法則）
4. **ラップタイム**: `demoTrajectory` 20点 × 200ms間隔＝1周約4000ms（約4秒）を基準に、`last_laptime` は周回ごとに `周回数 × 40ms`（`DEMO_LAP_SHORTEN_MS`）ずつ短縮される合成値（`test-mode.js` の `analysisOnFrame` 呼び出し部分）。実車のラップタイムを模したものではなく、PB更新判定など解析ロジックを検証するための内部シミュレーション値

## TEST MODEの用途

### 開発時

- UI変更の視覚確認（APEX Broadcast リデザイン後の見た目確認）
- レスポンシブレイアウトのテスト
- **DRIVE / ANALYSIS ビューの切替確認**（ヘッダーの操作メニューバー「表示」で走行用最小表示と全パネル表示を切替）
- 新機能の動作確認（距離基準ラップ解析チャート等）

### デモ・プレゼンテーション

- ダッシュボードの機能紹介
- UIデザインの共有
- スクリーンショット撮影

### トラブルシューティング

- PS5接続問題の切り分け
- ダッシュボード単体の動作確認

## エラーハンドリング

TEST MODEには以下のエラーハンドリングが実装されています：

- **トラジェクトリ検証**: デモトラジェクトリの妥当性チェック
- **要素存在確認**: DOM要素の存在確認
- **チャート検証**: チャートデータの整合性チェック
- **安全な実行**: すべての操作をtry-catchで保護

エラーが発生してもTEST MODEは継続され、コンソールに詳細なログが出力されます。

## TEST MODEの制限

- 実際の車両データとは異なります
- ラップタイムの記録はされません
- コース推定機能は使用されません（デモ軌跡データは内部計算専用）
- データ保存機能は動作しません

## PS5接続との切り替え

TEST MODE中にPS5からテレメトリデータを受信すると、自動的に実データに切り替わります。

## 関連ファイル

- `index.html`: TEST MODEボタンのHTML構造
- `test-mode.js`: テストモードのロジック（デモデータ生成・開始・停止）
- `ui_components.js`: DOM要素キャッシュ（テストモードが使用）
- `steer-response.js`: STEER RESPONSE 描画（`updateSteerResponse` をテストモードのデモ操舵が駆動）
- `course_database.json`: コース情報とTEST MODE設定

## トラブルシューティング

### TEST MODEボタンが反応しない

- ブラウザをリロード (F5)
- JavaScriptコンソールでエラーを確認 (F12)
- WebSocket接続が確立されているか確認

### デモデータが表示されない

- WebSocket接続が確立されているか確認
- コンソールログでエラーを確認
- ブラウザのコンソールで`[TEST_MODE]`タグのログを確認

### チャートが更新されない

- uPlotライブラリが正しく読み込まれているか確認
- ブラウザをリロード
