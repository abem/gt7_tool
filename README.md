# GT7 Telemetry Dashboard

グランツーリスモ7 (GT7) のリアルタイム・テレメトリダッシュボードです。

![Dashboard Screenshot](docs/images/dashboard-screenshot.png)

## 概要

PS5/PS4 から送信されるテレメトリパケットを受信・Salsa20 復号し、WebSocket 経由でブラウザ上のダッシュボードにリアルタイム表示します。

### 主な機能

| カテゴリ | 内容 |
|----------|------|
| **速度・駆動系** | 速度、RPM、ギア（推奨ギア）、ステアリング角度、ギア比、トルクベクタリング、回生エネルギー |
| **ペダル入力** | スロットル / ブレーキ / クラッチの生値 + TCS/ABS 補正後の値 |
| **タイヤ** | 4 輪の温度（色分け）、サスペンション高さ、ホイール回転速度、タイヤ半径 |
| **車体姿勢** | 3D 姿勢可視化（pitch/yaw/roll）、加速度 G、車体加速度（sway/heave/surge） |
| **燃料** | 残量、タンク容量、1 周消費量、残り周回予測 |
| **コース** | 位置座標からのコース自動判定、走行軌跡の速度別色分けマップ、レースライン着色（スロットル/ブレーキ/コースト） |
| **ラップ** | ラップタイム記録、ベストラップ、距離基準ライブ・タイムデルタ、推定ラップタイム、セクタータイム |
| **解析** | 距離索引ラップ解析、リファレンス速度重畳チャート、レースエンジニア通知、グリップ状態、一貫性σ |
| **ビュー切替** | DRIVE（走行用最小表示: ギア/シフトライト/デルタ中心、F1 ディスプレイ流）と ANALYSIS（全パネル解析表示）をワンクリック切替 |
| **インフラ** | Docker Compose 対応、HTTPS/WSS、テストモード（PS5 不要で動作確認可能）、全パケット対応（A/B/~） |

## クイックスタート

### 必須環境

- **グランツーリスモ 7** (PS5 / PS4)
- **Docker** (20.10+) / **Docker Compose** (2.0+)
- PS5 と PC が同一ネットワーク上にあること

### セットアップ

```bash
# 1. クローン
git clone https://github.com/abem/gt7_tool.git
cd gt7_tool

# 2. PS5 の IP アドレスを設定
#    config.json の "ps5_ip" を PS5 の実際の IP に変更
nano config.json

# 3. 起動
docker compose up --build

# 4. ブラウザで開く（HTTPS。自己署名証明書のため初回は警告→「詳細設定」から続行）
#    https://localhost:8080
```

> **PS5 の IP 確認方法:** 設定 → ネットワーク → 接続状態の詳細 → IP アドレス

### テストモード

PS5 がなくてもデモデータで動作確認できます。ダッシュボード右上の **TEST MODE** ボタンを押すと、シミュレーションデータが流れます。詳細は [TEST MODE ガイド](docs/test-mode.md) を参照してください。

## ダッシュボードの表示項目

### DRIVE / ANALYSIS ビュー

実車テレメトリの「ドライバー表示」と「エンジニア解析」の分離ドクトリン（F1 ステアリングディスプレイ / MoTeC i2 の設計原則）に基づき、ヘッダー右上のボタンで 2 つのビューを切り替えられます。選択はブラウザに保存されます。

| ビュー | 用途 | 表示内容 |
|--------|------|----------|
| **DRIVE** | 走行中 | ギア（特大）+ シフトライト + 速度 + ライブデルタ（大型・緑/赤）を中心に、LAP / LAST / BEST / EST / 燃料残周回 / タイヤ状態色ブロックの 6 項目 + コースマップのみ。0.2 秒で読める |
| **ANALYSIS** | ピット/走行後 | 全パネル（チャート、3D 姿勢、タイヤ詳細、距離解析、ラップ履歴等）。従来表示 |

### 速度・ギア
- 現在速度 (km/h)、最高速度記録
- 現在ギア (R, 1-8)、推奨ギア、ステアリング角度
- RPM バー（レブリミッター・シフトポイント表示）

### ペダル入力
- **THR**: スロットル入力 + TCS 補正値
- **BRK**: ブレーキ入力 + ABS 補正値
- **CLT**: クラッチ位置

### 車体姿勢 (CAR ATTITUDE)
- ミッドシップスポーツカー風 3D モデルによる姿勢可視化
- Pitch / Roll / Yaw のリアルタイム表示
- 角速度（RATE）の表示

### タイヤ・サスペンション
4 輪（FL/FR/RL/RR）ごとの温度（色分け）、サスペンション高さ、ホイール回転速度、タイヤ半径

### 駆動系・燃料
- ギア比表示（現在ギアをハイライト）
- ブースト圧、油圧、トルクベクタリング、回生エネルギー
- 燃料残量、タンク容量、1 周消費量、残り周回予測

### コース・ラップ
- 走行軌跡を速度別に色分けしたコースマップ（レースライン着色モード: スロットル緑 / ブレーキ赤 / コースト青、速度ピーク▴・バレー▾マーカー）
- ラップタイム履歴、ベストラップ差分デルタ、セクタータイム

### ラップ解析（距離基準）
- **ライブ・タイムデルタ**: ベストラップと同一距離地点での通過タイム秒差をリアルタイム表示
- **推定ラップタイム**: 走行中の全周外挿タイム + 自己ベスト更新判定
- **リファレンス速度重畳**: 距離軸チャートに現在ラップ + ベストラップの速度を重ねて表示
- **レースエンジニア通知**: 新トップスピード / PB 更新 / 燃料残量 (50-20-10%) / 残り周回 / ファイナルラップを通知フィードに表示
- **グリップ状態・一貫性σ**: タイヤグリップの状態表示とラップタイムのばらつき指標

## 構成

### バックエンド (Python)
| ファイル | 役割 |
|----------|------|
| `main.py` | エントリーポイント。HTTP/WebSocket サーバ (asyncio + aiohttp)。`telemetry_supervisor` による受信タスクの監視・再起動、`on_cleanup` による明示的シャットダウン |
| `telemetry.py` | PS5 との UDP 通信（`asyncio.DatagramProtocol` ベースの非同期受信・ハートビート送信） |
| `decoder.py` | Salsa20 暗号化パケットの復号・解析。Packet A/B/~ 対応、XOR 自動フォールバック |

### フロントエンド (HTML/CSS/JS)
| ファイル | 役割 |
|----------|------|
| `index.html` | ダッシュボードの HTML 構造 |
| `styles.css` | スタイル定義（APEX Broadcast デザインシステム、レスポンシブ対応） |
| `constants.js` | 配色パレット・チャート/マップ共通定数 |
| `ui_components.js` | 定数・設定・ユーティリティ・DOM キャッシュ |
| `charts.js` | uPlot チャート・加速度チャート・距離軸解析チャート管理 |
| `car-3d.js` | 3D 車両モデル・姿勢可視化 (Three.js) |
| `course-map.js` | コースマップ Canvas 描画・レースライン着色 |
| `websocket.js` | WebSocket 接続・テレメトリデータ処理 |
| `lap-manager.js` | ラップタイム記録・履歴管理 |
| `telemetry-analysis.js` | 距離基準ラップ解析（タイムデルタ、推定ラップ、エンジニア通知等） |
| `drive-view.js` | DRIVE ビュー（走行用最小表示）の切替・更新 |
| `test-mode.js` | テストモード（デモデータ生成） |
| `app.js` | エントリーポイント（各モジュール初期化） |

### 設定・データ
| ファイル | 役割 |
|----------|------|
| `config.json` | ネットワーク設定（PS5 IP、ポート、SSL証明書パス等） |
| `.env` / `.env.example` | 環境変数による設定上書き（env優先・config.jsonフォールバック） |
| `packet_def.json` | テレメトリデータのオフセット・型定義（参照用） |
| `course_database.json` | コース座標範囲データベース |

### テスト・スクリプト
| ファイル/ディレクトリ | 役割 |
|----------------------|------|
| `tests/test_decoder.py` | Salsa20 復号・XORフォールバック・parse・CourseEstimator の回帰テスト（pytest・12件） |
| `tests/test_course_detection.py` | コース推定ロジックの検証・DB再生成（`--regenerate`） |
| `scripts/` | 開発・検証用スクリプト群（debug/verify/capture系・gitignore対象・本番不含） |

## トラブルシューティング

| 症状 | 原因 | 対処 |
|------|------|------|
| ブラウザで接続できない | `http://` でアクセスしている | サーバーは HTTPS 専用。**`https://`** でアクセスし、自己署名証明書の警告は「詳細設定」から続行 |
| 証明書の警告が出る | 自己署名証明書のため正常な動作 | 「詳細設定」→「アクセスする（安全ではありません）」で続行（ローカル利用のため問題なし） |
| "Client connected" が出ない | PS5 IP が間違い / テレメトリ送信が OFF | `.env` または `config.json` の `ps5_ip` を確認、PS5 設定でテレメトリ送信を ON |
| データが表示されない | GT7 未起動 / レース画面でない | GT7 を起動しレースまたはタイムアタックで走行開始 |
| ネットワークエラー | コンテナの状態異常 | `docker compose down && docker compose up --build` |
| `Address already in use` | 別の GT7 ツールが同一ポートを使用中 | `docker ps` で他のコンテナ（例: gt7_tool_car_attitude2）を確認し停止、または `.env` でポート変更 |

詳細は [トラブルシューティング](docs/common-issues.md) を参照してください。

## カスタマイズ

### テレメトリ項目の追加
1. GT7 テレメトリ仕様でオフセットを確認
2. `decoder.py` の `_extract_fields()` にフィールドを追加
3. `index.html` に HTML 要素を追加、`websocket.js` で表示処理を実装
4. `docker compose restart` で反映

### UI の変更
`styles.css` でスタイルを変更。機能追加は対応する JS モジュールを編集。

### コースデータベースの更新

`tests/test_course_detection.py --regenerate` を実行すると、`gt7data/` を解析して `course_database.json` を再生成します（破壊的・上書き）。

> ※ **ホスト側（リポジトリルート）で実行してください**。コンテナ内には `tests/` がコピーされていないため `docker compose exec` では実行できません。`pycryptodome` が必要で、`decoder` モジュールを見つけるためリポジトリルートを `PYTHONPATH` に含める必要があります。

```bash
# ホスト側のリポジトリルートで実行
PYTHONPATH=. python3 tests/test_course_detection.py --regenerate
```

または `course_database.json` に手動でコース情報を追加。

### 2026-07-08 — バックエンド安定化と構成整理
- **Salsa20 復号の致命的バグ修正** — `decoder.py` の `_try_decrypt` で `from Crypto.Cipher import Salsa20` の遅延 import が未実装で、5日間・152万件の復号失敗が発生していたのを修正。PS5 実機テレメトリが復活

### 2026-07-02
- **DRIVE / ANALYSIS ビュー分離** — 実車テレメトリ（F1 ディスプレイ / MoTeC i2 / SimHub）の設計原則を調査し、走行用の最小表示 DRIVE ビューを新設。ギア特大表示 + 大型ライブデルタ + 6 項目ストリップ + コースマップのみに絞り、解析系パネルは ANALYSIS ビューに分離。ヘッダーのボタンで切替、選択は永続化
